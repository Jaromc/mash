Detailed Description

This is a singleton class and is only valid after the device has been created. It should be accesed from MashMemoryManager::Instance. The functions here don't normally need to be accessed directly. Instead, macros can be used from MashMemory.h.

All dynamic memory allocations should pass through here. The manager will log debug information about memory allocations so that memory leaks can be traced. Then it passes the actual memory allocation on to the custom allocator that is created by the user.
