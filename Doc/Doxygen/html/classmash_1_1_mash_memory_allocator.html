Detailed Description

Used for allocating dynamic memory. The user must provide one of these at load time.

Custom allocators can be created for memory pools or specialised operations.

The memory categories can be used to determine which memory pool the memory should be allocated from. These don't have to be considered though, a simple implimentation would simply allocate all memory from the one pool.

The allocate and deallocate functions are not normally acessed directly from here, instead go through the memory manager. The memory manager also logs memory debug info, so this class does not need to impliment that.
